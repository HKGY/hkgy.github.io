Basic Usage

gcc SOURCE_FILE_NAME -o OUTPUT_FILENAME

Something else

If I have a source named Hello_World.c,and I want to compile it and spawn executable file test,I need to run:

gcc Hello_World.c -o test

If any functions in math.h are used,-lm compile flag will be needed.For example,

gcc Math.c -o test -lm

If I want only to compile the file and get the object file,use -c option.-o can also be used to set the output file name. You should use .c as the extra file name,not .cpp.Beacuse GCC see files with .c extra file name as C Source and .cpp as C++ source. If you want to compile a CPP file,please use g++ instead of gcc. If there are more than one source file,list them in the command. e.g.:

gcc a.c b.c c.c d.c e.c -o test

If you want to debug the executable file,please use -g option and -O0 option.The first one lets GCC add symbol infomation to the result file, and the second one disables GCC optmizing to get a better experience while debugging. e.g.:

gcc bugged_source.c -o test -g -O0

You can use -Wall -Werror -pedantic to let gcc warn you the problems in your code.This can avoid many problems.
